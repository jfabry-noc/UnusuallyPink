It’s Always DNS

2020-08-30 :: John Fabry
#cradlepoint  #dns  #router 
There’s a saying among system administrators: It’s always DNS. Meaning that whenever there’s an issue, DNS is likely the culprit. This morning that adage proved itself yet again. My home network is currently running off of a Cradlepoint router. Cradlepoint’s specialty is making routers that can leverage LTE, so my router is configured to use my home ISP as the primary WAN link, but it will fail over to a cellular connection if my home ISP is unavailable.
